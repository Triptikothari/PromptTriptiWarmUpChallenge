# Intelligent Learning Assistant - Behavioral Analytics Plan

Based on the updated `code-guideline.md`, the platform needs to evolve from a "static-setting" assistant to a dynamic one. The assistant must intelligently analyze if the user is a **fast or slow learner**, how **deep they dive into topics**, and determine their language/understanding naturally based on history.

This document outlines the architectural changes required to implement this behavioral analysis.

## User Review Required

> [!IMPORTANT]
> **AI Analysis Approach:** Since we don't have a real LLM API hooked up yet to naturally infer "depth" or "pace", I will build a rule-based algorithm in our Django backend to calculate this. Are you happy with a rule-based approach for this prototype? (e.g., if a user replies "No" many times, they are classified as a "slow learner"; if they ask detailed follow-ups, they are classified as "deep divers").

## Proposed Changes

### 1. Database Model Expansions (`backend/api/models.py`)
To track the metrics that define a user's learning style, we need to expand our UserProfile and LearningHistory.
*   **Modify `UserProfile`**:
    *   Add `learning_pace` (e.g., "fast", "slow" - defaults to "moderate")
    *   Add `depth_preference` (e.g., "surface-level", "deep-dive")
*   **Modify `LearningHistory`**:
    *   Add `interaction_count`: the number of back-and-forth messages it took the user to understand this concept.

### 2. Analytical Engine Update (`backend/api/views.py`)
We will rewrite the `chat_view` logic to do the following:
*   **Pace Detection**: Every time the user replies "No" or asks for clarification on a topic, we increment the `interaction_count`. If average interaction counts exceed a threshold, we update their Profile `learning_pace` to "slow learner".
*   **Depth Detection**: If the user says "Yes, I understand!" very quickly, we categorize them as "surface-level". If they ask a detailed follow-up question, we classify them as a "deep-diver".
*   **Prompt Adjustment**: The text generated by the backend will automatically adjust based on these metrics. For a slow learner, the text will be shorter and use simpler analogies. For a deep diver, it will automatically introduce advanced theoretical syntax.

### 3. Frontend UI Updates (`frontend/app.jsx`)
*   Instead of the user manually selecting their "Level", the sidebar will show a **"Behavioral Insights"** panel that displays what the AI has learned about their pace and depth preferences in real-time.

## Open Questions
